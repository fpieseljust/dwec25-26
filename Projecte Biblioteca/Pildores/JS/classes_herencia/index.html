    Classes i herència

    Classes i herència

    A JavaScript, les classes i l'herència són mecanismes que permeten organitzar i estructurar el codi de manera més neta i reutilitzable, imitant el comportament d'altres llenguatges de programació orientats a objectes (OOP). Tot i que JavaScript no és un llenguatge estrictament orientat a objectes, la introducció de les classes en ECMAScript 6 (ES6) va fer que la programació orientada a objectes fos més accessible i intuïtiva.

    1. Classes en JavaScript

    Una classe és una plantilla o un model per crear objectes. És una forma de definir els atributs (propietats) i mètodes (funcions) d'un objecte.

    Definir una classe:

    A JavaScript, es poden definir les classes utilitzant la paraula clau class.

    class Persona {
  // Constructor: inicialitza els valors de l'objecte
  constructor(nom, edat) {
    this.nom = nom; // propietat
    this.edat = edat; // propietat
  }

  // Mètode de la classe
  saludar() {
    console.log(`Hola, em dic ${this.nom} i tinc ${this.edat} anys.`);
  }
}

// Crear una instància de la classe
const persona1 = new Persona('Joan', 30);
persona1.saludar(); // Output: Hola, em dic Joan i tinc 30 anys.

    • constructor(): El mètode especial constructor es crida automàticament quan es crea una nova instància de la classe. S'hi defineixen els atributs o propietats inicials de l'objecte.

    • Mètodes: Els mètodes es defineixen a l'interior de la classe. Aquests són simplement funcions que actuen sobre les propietats de l'objecte.

    2. Instanciació d'objectes

    Per crear un objecte a partir d'una classe, es fa servir la paraula clau new:

    3. Herència en JavaScript

    L'herència permet crear una nova classe (subclasse) a partir d'una classe existent (superclasse). La subclasse heretarà totes les propietats i mètodes de la superclasse i pot afegir-hi les seves pròpies.

    Sintaxi d'herència:

    A JavaScript, per crear una subclasse, s'utilitza la paraula clau extends.

    class Treballador extends Persona {
  constructor(nom, edat, feina) {
    // Cridem el constructor de la superclasse (Persona)
    super(nom, edat);  // super() crida al constructor de la classe pare
    this.feina = feina; // nova propietat per a la subclasse
  }

  // Mètode de la subclasse
  treballar() {
    console.log(`${this.nom} està treballant com a ${this.feina}.`);
  }
}

const treballador1 = new Treballador('Pere', 40, 'enginyer');
treballador1.saludar(); // Output: Hola, em dic Pere i tinc 40 anys.
treballador1.traballar(); // Output: Pere està treballant com a enginyer.

    Explicació de l'herència:

    • extends: Permet a la subclasse heretar les propietats i mètodes de la classe pare.

    • super(): Es crida al constructor de la classe pare. És necessari per inicialitzar les propietats de la classe pare abans de definir les pròpies de la subclasse.

    • Subclasse: Pot afegir mètodes propis a més dels mètodes heretats de la classe pare.

    4. Sobreescriptura de mètodes

    A la subclasse, pots sobreescriure un mètode de la classe pare. Això significa que pots redefinir un mètode que ja existeix a la classe pare per canviar el seu comportament.

    class Treballador extends Persona {
  constructor(nom, edat, feina) {
    super(nom, edat);
    this.feina = feina;
  }

  // Sobreescriure el mètode saludar de la classe Persona
  saludar() {
    console.log(`Hola, em dic ${this.nom} i sóc ${this.feina}.`);
  }
}

const treballador2 = new Treballador('Anna', 35, 'metgessa');
treballador2.saludar(); // Output: Hola, em dic Anna i sóc metgessa.

    En aquest exemple, el mètode saludar a la subclasse Treballador reemplaça el mètode saludar de la classe Persona.

    5. Propietats i mètodes estàtics

    A més de les propietats i mètodes d'instància, també podem tenir propietats i mètodes estàtics. Aquests no estan associats a una instància de la classe, sinó a la classe mateixa.

    Propietats i mètodes estàtics:

    JavaScript

    class Matematica {
  static PI = 3.1416;  // Propietat estàtica

  static calcularAreaRadi( radi ) {
    return Matematica.PI * radi * radi;  // Mètode estàtic
  }
}

console.log(Matematica.PI);  // Output: 3.1416
console.log(Matematica.calcularAreaRadi(5));  // Output: 78.54
    • static: S'usa per declarar mètodes o propietats que pertanyen a la classe i no a les instàncies de la classe.

    class Persona {
  constructor(nom, edat) {
    this._nom = nom;
    this._edat = edat;
  }

  // Getter per obtenir el nom
  get nom() {
    return this._nom;
  }

  // Setter per establir el nom
  set nom(nouNom) {
    this._nom = nouNom;
  }

  // Getter per obtenir l'edat
  get edat() {
    return this._edat;
  }

  // Setter per establir l'edat
  set edat(novaEdat) {
    if (novaEdat >= 0) {
      this._edat = novaEdat;
    } else {
      console.log('L\'edat no pot ser negativa!');
    }
  }
}

const persona3 = new Persona('Lluis', 28);
console.log(persona3.nom); // Output: Lluis
persona3.nom = 'Albert'; // Estableix un nou nom
console.log(persona3.nom); // Output: Albert

persona3.edat = -5; // Intentant establir una edat negativa

    Els getters i setters et permeten controlar l'accés i la validació de les propietats d'una classe.

    Resum

    • Classes: Són plantilles per crear objectes amb propietats i mètodes. S'utilitza class per definir-les.
    • Herència: Permet crear subclasses que hereten propietats i mètodes de les classes pare. S'utilitza extends per crear una subclasse.
    • Sobreescriptura de mètodes: Les subclasses poden redefinir els mètodes heretats.
    • Mètodes estàtics: Pertanyen a la classe en lloc de les instàncies.
    • Getters i Setters: Permeten accedir i modificar propietats de manera controlada.

    L'ús de classes i herència facilita l'estructuració de codi en JavaScript, fent-lo més modular, reutilitzable i net.
