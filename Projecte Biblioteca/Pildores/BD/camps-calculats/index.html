    Camps calculats

    Camps o valors calculats

    En bases de dades relacionals, els valors calculats (o camps calculats) no es guarden físicament a la taula, sinó que es generen a través de consultes cada vegada que es necessiten.

    Contextualitzant al nostre projecte de biblioteca, per calcular el nombre de llibres (o recursos) prestats a un soci, utilitzarem una consulta que compti els préstecs actius (aquells on la data_retorn sigui NULL, indicant que el recurs no ha estat retornat encara).

    Com calcular el nombre de llibres prestats

    La taula Prestecs ja conté la relació entre Socis i Recursos mitjançant les claus soci_id i recurso_id, així que podem obtenir el nombre de llibres prestats a un soci mitjançant una consulta.

    Exemple de consulta per calcular el nombre de llibres prestats:

    SQL
    1
2
3
    SELECT COUNT(*) AS llibres_prestats
FROM Prestecs
WHERE soci_id = 1 AND data_retorn IS NULL;
    • COUNT(*): Aquesta funció comptarà totes les files que compleixen amb la condició.
    • soci_id = 1: Substitueix 1 per l'ID del soci per al qual vols saber el nombre de llibres prestats.
    • data_retorn IS NULL: Filtra els préstecs que no han estat retornats, és a dir, aquells que encara estan actius.

    Camps calculats mitjançant una vista

    Una altra opció per tenir aquests càlculs disponibles sense haver de fer la consulta cada vegada és crear una vista (view), que és una consulta predefinida que es comporta com si fos una taula.

    Creació de la vista LlibresPrestats per obtenir el nombre de llibres prestats a cada soci:

    SQL
    1
2
3
4
5
    CREATE VIEW LlibresPrestats AS
SELECT p.soci_id, COUNT(*) AS llibres_prestats
FROM Prestecs p
WHERE p.data_retorn IS NULL
GROUP BY p.soci_id;

    Aquesta vista LlibresPrestats et permetrà consultar fàcilment quants llibres té cada soci en préstec:

    SQL
    1
    SELECT * FROM LlibresPrestats WHERE soci_id = 1;

    Aquesta consulta et retornarà el nombre de llibres que el soci amb id = 1 té actualment en préstec.

    Com funciona una vista?

    Una vista és una consulta SQL que es guarda al sistema de bases de dades i es pot utilitzar com si fos una taula. No emmagatzema les dades, sinó que les calcula cada vegada que es consulta. Això et permet obtenir informació dinàmica sense duplicar dades.

    Alternativa sense vista: Càlcul en temps real

    Si no vols crear una vista, sempre pots calcular el nombre de llibres prestats en temps real a través d'una consulta directa cada vegada que ho necessitis. Però la vista és una manera de fer-ho més neta i reutilitzable, sense haver de repetir la mateixa consulta en diverses parts del codi.

    Vista llibres prestats per soci

    Per obtenir la llista de llibres prestats per soci, utilitzant la taula Prestecs per calcular quins llibres estan actualment en préstec.

    Creació de la vista per obtenir la llista de llibres prestats per soci:

    SQL
    1
2
3
4
5
6
    CREATE VIEW LlibresPrestatsPerSoci AS
SELECT s.id AS soci_id, r.titol AS llibre, p.data_prestec
FROM Prestecs p
JOIN Recursos r ON p.recurso_id = r.id
JOIN Socis s ON p.soci_id = s.id
WHERE p.data_retorn IS NULL AND r.tipus = 'llibre';

    Explicació:

    • Prestecs p: Taula de préstecs, que relaciona socis i recursos.
    • JOIN Recursos r ON p.recurso_id = r.id: Uneix la taula Prestecs amb la taula Recursos per obtenir els títols dels recursos prestats.
    • JOIN Socis s ON p.soci_id = s.id: Uneix la taula Prestecs amb la taula Socis per obtenir els noms dels socis.
    • WHERE p.data_retorn IS NULL: Filtra els préstecs actius (on no s'ha retornat el recurs).
    • r.tipus = 'llibre': Filtra només els recursos que siguin de tipus llibre.
    • soci_id: L'ID del soci que té els llibres prestats.
    • llibre: El títol del llibre prestat.
    • data_prestec: La data en què es va realitzar el préstec.

    Consultes amb la vista

    1. Llistar tots els llibres prestats per soci:

    Per obtenir tots els llibres prestats per un soci en particular (per exemple, soci amb id = 1):

    SQL
    1
    SELECT * FROM LlibresPrestatsPerSoci WHERE soci_id = 1;

    Aquesta consulta et mostrarà tots els llibres prestats al soci amb id = 1.

    1. Llistar tots els llibres prestats (per a tots els socis):

    Per obtenir tots els llibres prestats a tots els socis (una llista general):

    SQL
    1
    SELECT * FROM LlibresPrestatsPerSoci;

    Aquesta consulta et mostrarà la llista de tots els llibres que estan actualment en préstec, juntament amb la informació del soci i la data de préstec.

    Avantatges de les vistes

    • Simplicitat: Amb aquesta vista, no has de repetir la mateixa consulta cada vegada que necessites saber quins llibres estan prestats a un soci.
    • Rendiment: Si la vista es consulta de manera eficient i està ben indexada, pots obtenir la informació més ràpidament que si fas les unions manualment a cada consulta.
    • Mantenibilitat: Si necessites modificar la lògica per calcular els llibres prestats (per exemple, afegir més filtres o relacionar altres taules), pots fer-ho fàcilment modificant només la vista.

    Resum

    • Camps calculats no es guarden físicament a la taula. En lloc d'això, els calculem a través de consultes SQL quan sigui necessari.
    • Pots crear una vista per tenir aquest càlcul disponible de manera fàcil, o bé calcular-lo directament en cada consulta.
